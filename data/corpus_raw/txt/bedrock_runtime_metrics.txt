The following table describes runtime invocation metrics provided by Amazon Bedrock.

1) Metric: Invocations
Unit: SampleCount
Description: Number of successful requests to the Converse, ConverseStream, InvokeModel, and InvokeModelWithResponseStream API operations.

2) Metric: InvocationLatency
Unit: MilliSeconds
Description: Latency of the invocations.

3) Metric: InvocationClientErrors
Unit: SampleCount
Description: Number of invocations that result in client-side errors.

4) Metric: InvocationServerErrors
Unit: SampleCount
Description: Number of invocations that result in AWS server-side errors.

5) Metric: InvocationThrottles
Unit: SampleCount
Description: Number of invocations that the system throttled. Throttled requests and other invocation errors don't count as either Invocations or Errors. The number of throttles depends on retry settings in the SDK. See "Retry behavior" in the AWS SDKs and Tools Reference Guide.

6) Metric: InputTokenCount
Unit: SampleCount
Description: Number of tokens in the input.

7) Metric: LegacyModelInvocations
Unit: SampleCount
Description: Number of invocations using legacy models.

8) Metric: OutputTokenCount
Unit: SampleCount
Description: Number of tokens in the output.

9) Metric: OutputImageCount
Unit: SampleCount
Description: Number of images in the output (only applicable for image generation models).
